## 🥤 1. 物理级杯内容器

### 功能说明

现在杯中的奶茶会按照你添加配料的顺序，从底到顶分层显示：

- **液体层** - 每种液体（茶、牛奶、椰奶）会显示为不同颜色的层
- **颜色混合** - 相邻层会自动混合颜色，看起来更真实
- **固体配料** - 珍珠、果冻、布丁等会叠加显示在液体上
- **动画效果** - 液面会随着添加配料逐渐上升

### 使用方法

```vue
<!-- 在App.vue中已自动使用 -->
<CupVisualEnhanced :contents="state.cupContents" />
```

### 示例

添加顺序：
1. 红茶 → 底部棕色层
2. 牛奶 → 中间白色层
3. 珍珠 → 顶部黑色圆球

最终显示：棕色-白色-珍珠的分层效果！

### 解锁配方

**方法**: 在游戏中成功制作饮料

- 当你杯中的配料组合匹配某个配方时
- 系统会自动提示"📖 新配方解锁: XXX!"
- 该配方会记录到图鉴中
- 显示首次制作时间和制作次数

### 配方匹配规则

- 配料种类和数量必须完全匹配
- 顺序不限（系统会自动排序比较）
- 示例：配方需要[红茶, 牛奶]，你添加[牛奶, 红茶]也能匹配

## 💾 5. 数据持久化

### 自动保存

游戏会自动保存以下数据到localStorage：

- ✅ 金币数量
- ✅ 解锁的配料和顾客
- ✅ 配方图鉴记录
- ✅ 成就解锁状态
- ✅ 每日登录记录
- ✅ 游戏统计数据

### 手动保存/加载

```typescript
import { saveGame, loadGame } from './composables/useSaveSystem'

// 保存
saveGame({
  coins: 1000,
  unlockedIngredients: ['boba', 'milk'],
  // ... 其他数据
})

// 加载
const saveData = loadGame()
console.log(saveData.coins) // 1000
```

### 导出/导入存档

```typescript
import { exportSave, importSave } from './composables/useSaveSystem'

// 导出为JSON字符串
const jsonString = exportSave()
console.log(jsonString) // 可以复制保存

// 从JSON导入
const success = importSave(jsonString)
if (success) {
  console.log('存档导入成功！')
}
```

## 🐛 故障排除

### 配方无法解锁

**问题**: 明明配料对了，但没提示解锁

**解决**:
1. 检查配料种类和数量是否完全匹配
2. 查看 `data/recipes.ts` 中的配方定义
3. 确认配料ID正确

### 连击特效不显示

**问题**: 连击了但没有粒子效果

**解决**:
1. 检查浏览器是否支持WebGL
2. 查看控制台是否有错误
3. 确认 `ParticleEffects.vue` 已正确导入

### 存档丢失

**问题**: 刷新页面后数据没了

**解决**:
1. 检查浏览器是否禁用了localStorage
2. 使用无痕模式会导致数据不保存
3. 定期使用 `exportSave()` 备份
